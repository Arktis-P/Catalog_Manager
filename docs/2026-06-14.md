## 1. 작업 큐 및 UI

### 캐릭터 수집 동시 실행 제한

- 시리즈별 Collect를 FIFO 큐로 관리
- 기본 **동시 2개** (Settings에서 1~5 조정)
- 완료 시 다음 시리즈 자동 시작
- 같은 시리즈는 중복 job 방지

### GlobalTaskBar 개선

- job당 한 줄 컴팩트 레이아웃
- `app-top` sticky로 **탭 이동·스크롤 시에도 상단 고정**
- Collect / Appearance job 구분 표시

### Settings 탭

- `danbooru_collect_max_concurrent` (1~5)
- 캐릭터 수집과 외형 추출 **공유 큐**

## 2. Phase 1 — 외형 태그 추출

### Danbooru related tags (`pybooru.tag_related`)

- 시리즈별 **Appearance** 버튼 (캐릭터 수집 후 활성)
- 추출 필드: `multi_color_hair`, `hair_color`, `hair_shape`, `eye_color`, `feature_tags`, `gender`

### 추출 규칙 (요약)

| 필드 | 규칙 |
|------|------|
| multi_color_hair | streaked → gradient → colored_inner 우선, 없으면 multicolored; streaked 시 색 streak 태그 추가 |
| hair_color | 허용 12색 중 상위 5개 |
| hair_shape | 길이 1 + 스타일 1 |
| eye_color | 1개 또는 heterochromia 시 2개 |
| gender | 1girl/1boy 등; no_humans·animal·monster·creature → `no_humans` |

### Catalog / Review 분리

- 수집 직후: **Review**에서 draft 확인
- `appearance_confirmed=True` 후: **Catalog**에만 표시
- `generation_prompt`: `{{character, [[hair color, ...]]}}` (hair + multi_color만)

### Appearance 재실행

- 전 캐릭터 **덮어쓰기**, `appearance_confirmed` 리셋 → Review 재확인

---

## 3. Series 상태 표시

- Appearance 완료 시 status **`tagged`**
- status 열 **한 개만** (우선순위: workflow → tagged → collected → collecting → pending)
- `all collected` 이중 배지 제거

---

## 4. 버그 수정·안정성

| 이슈 | 조치 |
|------|------|
| `launch_desktop.bat` 백엔드 기동 실패 | `settings` 라우터 import가 `app.config.settings` 가림 → `settings_router`로 분리 |
| Danbooru `related_tag` 500 | 429/500 등 **최대 3회 재시도** + rate limit 안내 |
| 오류 메시지 복사 불가 | error/task bar/테이블 등 **텍스트 선택 가능** (`user-select`) |

**API 확인:** `related_tag.json?category=0` 정상; 연속 호출·대량 수집 시 500 빈번 → rate limit 대응 필요.
