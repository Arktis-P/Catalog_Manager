# 2026-06-15 개발 일지 — Catalogue Manager

## 개요

Touhou 등 대형 시리즈에서 **크로스오버 캐릭터 오염** 문제를 계기로, 캐릭터 수집을 **위키/list 페이지 우선**으로 전환하고, **related tags copyright 검증**·**생성 게이트**·**데이터 초기화**까지 진행한 날.

---

## 1. 문제 정의

- `touhou` 태그 post 스캔 시 `hatsune_miku` 등 **타 시리즈 캐릭터**가 후보에 포함됨
- `character_tag + series_tag` post_count > 0 만으로는 크로스오버 일러스트를 걸러내기 어려움
- `*_(touhou)` suffix 패턴 검증은 **실효성 낮음** (`hakurei_reimu` 등 suffix 미사용 태그가 대부분)

---

## 2. 위키 기반 캐릭터 수집 (Phase 1)

### 방식

- **HTML 크롤링 없음** — Danbooru 공식 API `wiki_pages.json` + `tags.json`(category 분류)
- DText `[[tag]]`, `[[label|tag]]` 링크 파싱 (`wiki_dtext.py`)
- `list_of_{series}_characters` 타이틀 탐색·검색 후 character(category=4)만 수집
- 수집 출처: `from_wiki`, `from_list_page` 플래그

### 하위 시리즈 hub 처리

- 시리즈 위키에 **캐릭터 대신 copyright(하위 게임) 링크**만 있는 경우 (예: Touhou)
- DB에 해당 `series_tag`가 있으면 → 하위 시리즈 각각 위키 수집 → 캐릭터 저장 → **상위 시리즈로 자동 병합**
- DB에 없는 하위 태그는 `skipped_sub_series`로 기록

### Legacy fallback

- 위키·하위 병합 후에도 **캐릭터 0명**이면 기존 방식(post 스캔 + `*_ (series)` 패턴)으로 재시도
- fallback 시 post 스캔 **강제 활성** (`post_supplement=True`)
- API 응답 `used_legacy_fallback` 플래그 추가

### 설정 (기본값)

| 설정 | 기본 | 의미 |
|------|------|------|
| `danbooru_character_wiki_collect` | true | Collect 시 위키 경로 우선 |
| `danbooru_character_post_supplement` | false | 위키 경로에서 post 스캔 보조 끔 |
| `danbooru_character_legacy_fallback` | true | 위키 실패 시 legacy 수집 |

### 주요 코드

- `backend/app/integrations/danbooru/wiki_dtext.py`
- `backend/app/integrations/danbooru/wiki_character_collector.py`
- `backend/app/services/character_service.py` (`_collect_for_series_wiki`, `_maybe_legacy_fallback`)
- `backend/app/integrations/danbooru/client.py` (`get_wiki_page`, `search_wiki_pages`)

## 3. 시리즈 소속 검증 (related tags)

### 규칙

- Appearance 추출 시 `related_tag` 응답에서 **copyright(category=3)** 만 추출
- 현재 시리즈(및 병합된 하위·`source_series`)보다 **다른 copyright가 2배 이상** 연관되면 `needs_check`
- `needs_check_reason`에 `Series membership: …` 접두사로 사유 저장

### UI

- **Characters 팝업**: Status 필터 `needs_check only`
- **Generation**: 기본 목록에서 `needs_check` **제외**; 「needs_check 캐릭터만 보기」로 검토 전용 목록 (생성 버튼 비활성)
- 큐 생성 시 `needs_check` 캐릭터는 선택해도 **스킵**

### 주요 코드

- `backend/app/integrations/danbooru/series_membership.py`
- `backend/app/services/appearance_service.py`
- `frontend/src/components/SeriesCharactersModal.tsx`
- `frontend/src/pages/GenerationPage.tsx`
