# 카탈로그 매니저 사용법 가이드

> 이 문서는 카탈로그 매니저의 전체 자동화 흐름과 사용법을 쉽게 설명합니다.
> 처음 사용하는 분도 순서대로 따라가면 됩니다.

---

## 이 프로그램이 하는 일

쉽게 말하면, 이 프로그램은 아래 세 가지 일을 자동으로 도와줍니다.

1. **캐릭터 정보 모으기** — Danbooru에서 특정 시리즈의 캐릭터 목록과 머리색·눈색 같은 외형 정보를 자동 수집합니다.
2. **이미지 만들기** — NAIA(NovelAI)를 이용해 캐릭터 이미지를 자동으로 생성합니다.
3. **이미지 검사하기** — 만들어진 이미지가 실제로 해당 캐릭터처럼 보이는지, 눈·손 디테일이 제대로 나왔는지 자동으로 판단합니다.

이 세 가지를 사람이 직접 하면 시간이 엄청 많이 걸리는데, 자동화 덕분에 최소한의 확인만 하면 됩니다.

---

## 전체 워크플로우 한눈에 보기

```
[시리즈 선택]
    ↓
[캐릭터 수집] ← Danbooru에서 자동으로 캐릭터 목록 가져오기
    ↓
[외형 태그 추출] ← 각 캐릭터의 머리색·눈색 등 정보 자동 수집
    ↓
[외형 태그 확인] ← 사람이 한 번 확인 (잘못된 정보 수정)
    ↓
[이미지 생성] ← NAIA로 자동 이미지 생성 + 자동 품질 검사
    ↓ (검사 실패 시 자동으로 레벨 올려 재생성)
    ↓ (완료 후 자동으로 커버 이미지 선정)
    ↓
[이미지 검수] ← 사람이 결과 확인 · 평점 입력
    ↓
[카탈로그 완성!]
```

---

## 준비 단계: 처음 한 번만 하면 됩니다

### 1. HF Token 발급

"HF Token"은 AI 이미지 분석 서비스(Hugging Face)를 사용하기 위한 비밀번호 같은 것입니다.
이걸 설정하면 이미지 생성 후 "이 이미지가 실제로 그 캐릭터인지" 자동으로 확인해 줍니다.

**발급 방법:**
1. [huggingface.co](https://huggingface.co) 접속 → 회원가입 또는 로그인
2. 오른쪽 위 프로필 사진 클릭 → **Settings** 클릭
3. 왼쪽 메뉴에서 **Access Tokens** 클릭
4. **New token** 버튼 클릭 → 이름 아무거나 입력 → **Read** 권한 선택 → **Create** 클릭
5. 생성된 토큰(`hf_...`로 시작하는 긴 문자열) 복사

**Settings 페이지에 입력:**
1. 프로그램 상단 메뉴 → **Settings** 클릭
2. "Hugging Face Token (WD 자동 태깅)" 항목에 복사한 토큰 붙여넣기
3. **Save** 버튼 클릭

> **HF Token이 없어도 됩니다!** 없으면 캐릭터 AI 판별은 안 되지만, 눈·손 품질 검사는 계속 작동합니다.

---

## 1단계: 시리즈 등록 및 캐릭터 수집

### 시리즈 등록하기

1. 상단 메뉴 → **Series** 클릭
2. 오른쪽 위 **+ 시리즈 추가** 버튼 클릭
3. **series_tag** 칸에 Danbooru에서 쓰는 시리즈 태그 입력
   - 예: `genshin_impact`, `blue_archive`, `hololive`
4. **저장** 클릭

### 캐릭터 자동 수집

1. 방금 만든 시리즈의 행에서 **수집 시작** 버튼 클릭
2. 프로그램이 Danbooru에서 자동으로 캐릭터 목록을 가져옵니다
3. 완료될 때까지 기다립니다 (수십 명이면 수십 초)

> **포스트 수 필터**: Settings에서 "캐릭터 최소 포스트 수"를 설정하면 인지도가 너무 낮은 캐릭터는 자동으로 건너뜁니다. 기본값 10~20 추천.

---

## 2단계: 외형 태그 추출 및 확인

### 외형 태그란?

각 캐릭터의 "머리색", "눈색", "특징 태그"(안대, 뿔, 리본 등)를 말합니다.
이 정보가 있어야 이미지 생성 프롬프트를 정확하게 만들 수 있습니다.

### 자동 추출

1. Series 목록에서 해당 시리즈의 **외형 추출 시작** 버튼 클릭
2. Danbooru에서 각 캐릭터의 포스트를 분석해 통계적으로 외형 정보를 추출합니다
3. 완료 후 **Review → Appearance** 탭에서 결과 확인

### 외형 태그 검토 (Review → Appearance 탭)

1. 상단 메뉴 → **Review** 클릭 → **Appearance** 탭 선택
2. 캐릭터 목록이 표시됩니다
3. 각 캐릭터를 클릭해서 머리색·눈색 등이 맞는지 확인
4. 잘못된 부분이 있으면 수정 후 **Confirm** 버튼 클릭
5. Confirm된 캐릭터만 이미지 생성 대상이 됩니다

> **팁**: 확인이 너무 오래 걸릴 때는 Generation 화면의 "외형 태그 Confirm된 캐릭터만 포함" 체크를 해제하면 미확정 캐릭터도 포함해서 생성할 수 있습니다.

## 3단계: 이미지 생성

### Generation 페이지 이동

1. 상단 메뉴 → **Generation** 클릭
2. "NAIA 연결" 상태가 **연결됨** (초록색)인지 확인

### 생성 시작 순서

**① 시리즈 선택**
- "생성 대상" 아래 시리즈 드롭다운에서 원하는 시리즈 선택
- 선택하면 캐릭터 목록과 함께 **추천 Lv 배지**가 표시됩니다

**② Prompt Level 선택**

| 레벨 | 내용 | 언제 쓰면 좋을까? |
|------|------|------------------|
| Level 1 | 캐릭터 태그만 사용 | 포스트가 매우 많은 인기 캐릭터 |
| Level 2 | 기본 캐릭터 코어 | 포스트 300~1000개 정도 |
| Level 3 | 머리·눈 형태 포함 | 포스트 적고 태그 풍부한 캐릭터 |
| Level 4 | feature tags 포함 | 특징이 특이한 캐릭터 |
| Level 5 | 최대 강도 | 생성이 계속 실패하는 경우 |

> **추천 레벨 배지**: 시리즈 선택 후 "추천 Lv2" 같은 배지가 표시됩니다.
> 배지를 **클릭**하면 해당 레벨로 바로 설정됩니다.
> 배지에 마우스를 올리면 "Lv1: 5명, Lv2: 3명, Lv3: 2명" 같은 분포도 확인할 수 있습니다.

**③ 생성 시작**
- **큐 미리보기** 버튼으로 생성될 내용 미리 확인 가능
- **이미지 생성 시작** 버튼 클릭!

### 생성 중에 일어나는 일 (전부 자동)

생성이 시작되면 아래 순서로 자동 처리됩니다:

```
각 캐릭터마다:
  1. NAIA에서 이미지 생성 (기본 2장)
  2. WD 태거로 이미지 분석
     → "이 이미지의 캐릭터가 맞나?"
     → "머리색·눈색이 맞나?"
     → "성별이 맞나?"
  3. 눈·손 품질 검사
     → 눈이 균형있게 그려졌나?
     → 손가락이 제대로 나왔나?
  4. auto_status 판정:
     → pass: 통과 ✅
     → warning: 확인 필요 ⚠️
     → reject_candidate: 실패 ❌
  5. 만약 모든 이미지가 ❌이면?
     → 자동으로 Level +1에서 1장 더 생성!
     → 여전히 ❌이면 Level +1 한 번 더 시도
     → (최대 2번 자동 재시도)
  6. 커버 이미지 자동 선정
     → ✅ 이미지 중 cover_score 최고점을 커버로 설정
```

### 진행 상황 확인

생성 중에는 하단에 진행 패널이 표시됩니다:

- **생성 중**: `캐릭터명 생성 중 (5/20)` — 현재 몇 번째 캐릭터인지
- **에스컬레이션**: `캐릭터명 Level 3 재생성 (1/2)` — 자동 레벨 상승 중
- **커버 선정**: `커버 이미지 자동 선택 중...` — 마지막 단계
- 완료 후: `완료: 40장 저장 · pass 32 / warning 5 / reject 3`

## 4단계: 검수 (Review)

### Catalog Review 페이지

1. 상단 메뉴 → **Review** 클릭 → **Catalog Review** 탭 선택
2. 시리즈를 선택하면 캐릭터 목록이 표시됩니다

### 트리아지 필터 활용 (핵심!)

이미지 생성 후 Filter 드롭다운에 새 옵션이 생겼습니다:

| 필터 | 의미 | 할 일 |
|------|------|--------|
| ⚡ **자동 통과 (WD pass)** | AI가 "이 이미지 OK"라고 판단 + 커버 자동 선정 | 레이팅만 입력하면 완료! |
| 🔍 **확인 필요 (warning)** | AI가 약간 불확실하게 판단 | 이미지를 보고 직접 판단 |
| 🔄 **재생성 필요 (all reject)** | 모든 이미지가 AI 검사 실패 | `r` 키로 재생성 |
| Pending | 아직 검수 안 된 전체 목록 | 기본 보기 |

#### 권장 검수 순서

**1단계: 먼저 "자동 통과"부터**

Filter → "⚡ 자동 통과" 선택

이 캐릭터들은 커버도 자동으로 정해져 있어서, 평점만 입력하면 끝납니다!

- 숫자키 **1~6**: 평점 입력 (재입력 시 해제)
  - 6: 매우 좋아함
  - 5~4: 좋아함
  - 3: 보통 (잠재)
  - 2~1: 별로
  - 0: 생성 불가 (이미지 전부 삭제)
  - -: 대상 아님 (이 캐릭터는 카탈로그에 넣지 않음)
- **Enter**: 검수 완료 후 다음 캐릭터로

**2단계: "확인 필요" 처리**

Filter → "🔍 확인 필요" 선택

이미지를 보면서 직접 판단합니다:
- **← →** 화살표로 이미지 넘기기
- **Space**: 이미지 크게 보기
- 좋아 보이는 이미지를 클릭해서 커버로 선택
- 평점 입력 후 **Enter**

**3단계: "재생성 필요" 처리**

Filter → "🔄 재생성 필요" 선택

- **r** 키: 재생성 시작 (NAIA에서 새 이미지 만들기)
- 재생성 후 이미지 확인 → 평점 입력

## 자동화 기능 상세 설명

### WD 태거 (캐릭터 AI 판별)

WD 태거는 "Waifu Diffusion Tagger"의 약자로, 이미지를 보고 Danbooru 태그를 예측하는 AI입니다.
HF Token을 설정하면 Hugging Face 서버에서 실행되므로 내 컴퓨터에 AI 모델을 설치하지 않아도 됩니다.

**판별 항목:**
- 이 캐릭터 태그가 이미지에 있는가? (캐릭터 일치 여부)
- 머리색이 맞는가? (예: `brown_hair`)
- 눈색이 맞는가? (예: `blue_eyes`)
- 성별이 맞는가? (예: `1girl`)

### 눈·손 품질 검사

WD 태거와 별개로, 이미지 픽셀을 직접 분석합니다:

| 검사 항목 | 기준 | 설명 |
|-----------|------|------|
| 눈 선명도 | 18% 이상 | 눈이 흐릿하면 실패 |
| 눈 대칭성 | 0.55 이상 | 왼눈·오른눈이 비슷해야 함 |
| 눈 대비도 | 30 이상 | 눈 주변에 뚜렷한 경계선 있어야 함 |
| 손 선명도 | 22% 이상 | 손이 흐릿하면 실패 |
| 손가락 수 | 3~8개 | 너무 적거나 많으면 실패 |

> 손이 이미지에 없거나 가려진 경우 손 검사는 자동으로 건너뜁니다.

### auto_status 판정 기준

| 상태 | 의미 | 조건 |
|------|------|------|
| **pass** ✅ | 모든 검사 통과 | 캐릭터 일치 + 품질 양호 |
| **warning** ⚠️ | 부분 통과 | 캐릭터는 맞지만 품질 불확실 |
| **reject_candidate** ❌ | 검사 실패 | 캐릭터 불일치 또는 품질 나쁨 |

### 자동 레벨 에스컬레이션

```
캐릭터 A의 이미지 2장 모두 ❌ 판정됨
  → 자동으로 Level+1에서 1장 더 생성
  → 새 이미지가 ✅ → 에스컬레이션 완료
  → 새 이미지도 ❌ → Level+1 한 번 더 시도
  → 여전히 ❌ → "재생성 필요" 목록에 들어감
  (최대 2번 자동 재시도, Level 5 초과 없음)
```

이 덕분에 "자동 통과"·"확인 필요" 목록이 줄고, "재생성 필요" 목록만 수동으로 처리하면 됩니다.

### 커버 이미지 자동 선정

모든 이미지 생성이 끝나면 자동으로 각 캐릭터의 "가장 좋은 이미지"를 커버로 지정합니다:

1. WD pass 이미지가 있으면 그 중에서 선택
2. WD pass가 없으면 전체 이미지 중에서 선택
3. cover_score(캐릭터 일치도 × 0.5 + 머리색 × 0.15 + 눈색 × 0.15 + 성별 × 0.10 + 품질 × 0.10)가 가장 높은 이미지 선택

검수 페이지에서 커버가 이미 선정되어 있으므로, "자동 통과" 캐릭터는 레이팅만 입력하면 됩니다.

## 추천 레벨 계산 방법

캐릭터별로 아래 기준으로 레벨을 자동 추천합니다:

| 조건 | 추천 레벨 | 이유 |
|------|-----------|------|
| 포스트 수 1000개 이상 | Level 1 | NAIA가 이미 잘 알고 있음 |
| 포스트 수 300~999개 | Level 2 | 어느 정도 알고 있지만 힌트 필요 |
| 외형 태그 4개 이상 | Level 3 | 태그를 많이 넣어야 캐릭터가 나옴 |
| 외형 태그 2~3개 | Level 2 | 기본 정보만 제공 |
| 외형 태그 1개 이하 | Level 1 | 태그가 적으면 Level 높여도 도움 안 됨 |

배치 전체에서 **가장 높은 레벨**을 배지로 표시합니다.
(보수적 기준 — 모든 캐릭터를 커버하려면 가장 높은 레벨로 통일)

## 자주 묻는 질문 (FAQ)

**Q. HF Token을 설정하지 않으면 어떻게 되나요?**
A. 눈·손 품질 검사만 작동합니다. 캐릭터 일치 여부 확인은 안 되지만, 이미지 자체의 품질은 검사합니다. auto_status는 품질 기반으로만 판정됩니다.

**Q. 생성 중간에 취소할 수 있나요?**
A. 네, 진행 패널의 **취소** 버튼을 누르면 됩니다. 이미 생성된 이미지는 저장됩니다.

**Q. "자동 통과"인데 이미지가 마음에 안 들면?**
A. 해당 캐릭터를 선택 후 **r** 키를 누르면 재생성합니다. 또는 직접 다른 이미지를 클릭해 커버를 바꿀 수 있습니다.

**Q. 생성 이미지가 계속 "재생성 필요"로 나와요**
A. 다음을 확인해 보세요:
1. 외형 태그가 정확히 입력되어 있는지 확인 (Review → Appearance)
2. 더 높은 Prompt Level로 시도
3. 캐릭터 포스트 수가 너무 적으면 (20개 미만) NAIA가 잘 모를 수 있음

**Q. WD 모델을 바꾸고 싶어요**
A. Settings → "HF WD Model ID" 칸을 수정하세요. 기본값은 `SmilingWolf/wd-eva02-large-tagger-v3`입니다.

**Q. 시리즈당 이미지를 몇 장 생성하는 게 좋나요?**
A. 기본 2장을 권장합니다. 에스컬레이션으로 최대 4장(2장 + 재시도 2장)이 생성될 수 있습니다. Settings에서 "캐릭터당 생성 이미지 수"를 조정할 수 있습니다.
